# Revenue & Paid Users Dynamics Dashboard

This project presents a Tableau dashboard designed to analyze monthly revenue dynamics and user behavior in a gaming context. It was built as part of a data analytics course and demonstrates SQL data modeling, metric calculation, and dashboard visualization.

## 📊 Dashboard

🔗 [View the Tableau Dashboard](https://public.tableau.com/views/RevenuePaidUsersDynamicsDashboard/RevenueUserMetricsDashboard)

## 📁 Project Structure

- `sql/revenue_metrics_query.sql`: SQL query used to extract and transform data from the PostgreSQL database.
- `images/schema.png`: Database schema used for data modeling.
- `docs/project_description.md`: Full project description and requirements.
- `dashboard_link.txt`: Direct link to the Tableau dashboard.

## 📈 Metrics Included

The dashboard includes the following key metrics:

- Monthly Recurring Revenue (MRR)
- Paid Users
- Average Revenue Per Paid User (ARPPU)
- New Paid Users
- New MRR
- Churned Users
- Churn Rate
- Churned Revenue
- Revenue Churn Rate
- Expansion MRR
- Contraction MRR
- Customer Lifetime (LT)
- Customer Lifetime Value (LTV)

## 📌 Filters

The dashboard includes filters by:
- Date
- User Language
- User Age

## 📐 Visualizations

At least five charts are included, with two specifically showing factors behind changes in revenue and paid users month-over-month.

## 🧠 Technologies Used

- PostgreSQL
- SQL (CTEs, window functions)
- Tableau Public

## 📚 Purpose

This dashboard helps product managers monitor revenue trends and analyze user behavior to make data-driven decisions.
